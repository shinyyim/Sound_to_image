## Work Completed

### 1. Project Setup
- Configured `.claude/` folder (CLAUDE.md, settings, rules, skills)
- Created project directory structure (`src/`, `data/`, `outputs/`, `configs/`)
- Wrote `requirements.txt` with all dependencies
- Created `.env` for API key management
- Set up `.gitignore`

### 2. Audio Analysis Pipeline (Stages 1–3)
- **Stage 1 — Capture** (`src/capture/loader.py`): FOA audio loader with 4-channel validation and resampling
- **Stage 2 — Spatial Analysis** (`src/spatial_analysis/analyze.py`):
  - FOA directional cue extraction, spectrogram, onset density, loudness variance
  - Spectral entropy, band energy (low/mid/high), RT60 estimation, motion trajectory
- **Stage 3 — Audio Embedding** (`src/audio_embedding/embed.py`): LAION-CLAP embedding (512-dim)

### 3. Visualization
- **Static PNG Dashboard** (`src/spatial_analysis/visualize.py`): 8-panel analysis chart
- **Interactive HTML Dashboard** (`dashboard.html`):
  - Real-time visualization via Web Audio API (waveform, spectrum, radial, spectrogram, stereo field, loudness)
  - Live stats update (onset, DR, entropy, RT60, enclosure, L/M/H)
  - Balenciaga typeface + ultra-minimal monochrome UI
  - Design referenced from dialedweb.com

### 4. Metadata Prediction + LLM Interpretation (Stages 4–5)
- **Stage 4 — Metadata Predictor** (`src/metadata_predictor/predict.py`):
  - Rule-based environment classification (RT60, entropy, onset density combinations)
  - Source object inference, acoustic profiling, volatility index
- **Stage 5 — LLM Interpreter** (`src/llm_interpreter/interpret.py`):
  - Claude API integration — metadata → scene description + diffusion prompts
  - Prompt suffix: `shot on 35mm lens, f/2.8, Kodak Portra 400, cinematic lighting, 8k, highly detailed, physically based rendering`
  - Template-based fallback when API is unavailable

### 5. Image Generation (Stage 6)
- **Image Generation** (`src/image_generation/generate.py`):
  - Added GPT Image (`gpt-image-1`) support
  - DALL-E 3, Gemini (`gemini-2.5-flash-image`), Stability API, and placeholder modes
  - Front/Lateral/Aerial 3-view or single image generation

### 6. CLAP Environment Matching (Accuracy Improvement)
- **CLAP Environment Matching** (`src/audio_embedding/match_environment.py`):
  - Bank of 20 environment descriptions (forest, city, subway, beach, etc.)
  - Audio ↔ text cosine similarity comparison
  - Fixed NaN issue by switching to `get_audio_embedding_from_filelist`
  - Result: "SFERA POST Forest" → `river or stream in nature (0.416)`
- **YAMNet** (`src/spatial_analysis/classify.py`): Code written, disabled due to TensorFlow incompatibility with Python 3.14

### 7. Server (`server.py`)
- Local proxy server on port 8420
- `/api/chat` — Claude API proxy (keeps API key server-side)
- `/api/pipeline` — Runs Stages 4→5→6 sequentially
- `/api/upload` — Audio file upload → automatic analysis (librosa + CLAP)
- Timestamped output folders under `outputs/runs/`

### 8. Dashboard Features
- Auto-upload to server on file load → CLAP analysis
- [Predict] [Interpret] [Full-Image] pipeline buttons
- Custom prompt input → Generate (GPT Image)
- Image click → fullscreen modal
- Saved file path displayed in chat
